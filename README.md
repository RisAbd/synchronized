# synchronized

Инструмент, который показывает **оригинальный текст Корана синхронно со звучащим чтением** и авто-прокручивает его, чтобы человек мог следить/читать вместе с чтецом в оригинале.

## Зачем (кейс автора)

Слушаю чтение Корана в машине (YouTube / аудио). Часто одну и ту же суру. Хочется читать вместе с чтецом по оригиналу, но точных слов наизусть не помню — только примерно. Нужен крупный оригинальный текст перед глазами, который **сам прокручивается в такт аудио** (за рулём в экран не смотрю пристально — только скольжу взглядом).

## Два режима

- **A. Файл/ссылка (offline).** Даю ссылку на YouTube или аудиофайл → инструмент готовит синхронную «дорожку» текста и проигрывает с авто-прокруткой. Возможность прыгать по сурам/местам как по chapters.
- **B. Живое (live).** Приложение слушает поток в реальном времени (например, кто-то рядом читает), понимает, **какое это место** в Коране, открывает оригинальный текст и прокручивает его на лету. Даже если сам суру не знаешь — по паре слов находит место и «подкидывает» текст, дальше читаешь по нему.

## Ключевой принцип

Показываем **оригинальный** текст Корана (из `quran.db`), а не транскрипцию. Транскрипция (ASR) — лишь инструмент, чтобы **найти место** в оригинале и привязать тайминги. Дальше всё крутится по таймстемпам оригинала.

## Ограничения

- Локальный проект. Максимально лёгкий: без тяжёлых зависимостей и без тяжёлого GUI (никакого PyQt на старте). Плеер — сначала простая самодостаточная HTML-страница (открывается в т.ч. на телефоне).
- Сначала «лишь бы заработало», оптимизация потом.
- Модульность: независимые части с JSON-контрактами между ними, каждую можно пилить и тестировать отдельно.


Пути в коде — относительно расположения файла (`src/`), так что скрипты работают из любой директории.

## Сервис (Django)

Веб-сервис, куда кидаешь ссылку → фоновая обработка (ingest→asr→align) → плеер с
пословной подсветкой и авто-прокруткой. Список записей со статусами.

```bash
# dev (SQLite, обработка в потоке — без Redis):
cd service
python3 manage.py migrate
python3 manage.py seed_demo          # демо-запись Аль-Исра
python3 manage.py runserver 0.0.0.0:8000
# ASR на GPU: выставь LD_LIBRARY_PATH на cuDNN/cuBLAS (см. src/asr.py); CPU: SYNC_ASR_DEVICE=cpu

# стандартный стек (web + Postgres + Redis + Celery-worker):
docker compose up --build            # http://localhost:8000
```

> Тяжёлый ASR (GPU) удобнее гонять на хосте; в docker faster-whisper работает на CPU (медленно).

## Запуск (рабочий прототип)

```bash
# из файла (любой аудио/видео):
python3 src/run.py path/to/recitation.mp3

# из ссылки YouTube (см. оговорку ниже):
python3 src/run.py "https://www.youtube.com/watch?v=oM1e8fILtaM"
```

Конвейер: ingest → asr (faster-whisper, GPU) → align → плеер. Результат —
`work/<name>.player.html`; открой его в браузере (аудио лежит рядом). Крупный оригинальный
текст сам прокручивается и подсвечивает текущий аят в такт чтению.

Модули запускаются и по отдельности: `python3 src/quran.py`, `python3 src/asr.py <audio>`,
`python3 src/align.py <transcript>`, `python3 src/player.py <sync-map> <audio>`.

> **cuDNN:** перед ASR нужен `LD_LIBRARY_PATH` на pip-пути cuDNN/cuBLAS (`run.py` выставляет
> сам; вручную — см. `src/asr.py`).
>
> **YouTube:** актуальный `yt-dlp` пока упирается в анти-бот YouTube («not available on this
> app»). Обход — свежий yt-dlp либо `--cookies-from-browser`; для файла всё работает без оговорок.

## Веб-фронт + доступ другу (ngrok)

Есть веб-версия (лендинг со списком чтений + плеер), отдаётся лёгким сервером на stdlib
(с поддержкой HTTP Range — аудио перематывается/стримится).

```bash
# 1) подготовить запись для веба (один раз на запись):
python3 src/build_web.py --id aswailis-isra \
    --sync-map work/audio.sync-map.json --audio work/audio.mp3 \
    --title "Сура Аль-Исра (фрагмент)" --title-ar "سورة الإسراء" \
    --reciter "الشيخ يونس اسويلص"

# 2) поднять сервер:
python3 src/server.py        # http://localhost:8000

# 3) дать доступ другу:
ngrok http 8000              # кидаешь другу выданную https-ссылку
```

Друг открывает ссылку → выбирает запись → слушает, текст сам скроллится. Работает и на телефоне.
