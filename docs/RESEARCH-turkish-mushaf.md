## ⚠️ ЭМПИРИЧЕСКАЯ ПРОВЕРКА (11.07, после ресёрча) — ВЛАДЕЛЕЦ ПРАВ, ТЕКСТ ДРУГОЙ

Сверил Аль-Фатиху из диянетовского источника (`api.acikkuran.com/surah/1`, текст Diyanet)
с нашим Tanzil — **Unicode РЕАЛЬНО отличается** (слой dabt), а не только шрифт:

| | наш (Tanzil) | Diyanet |
|---|---|---|
| 1:1 «Аллах» | اللَّه (лям+шадда+**фатха** 064E) | اللّٰه (лям+шадда+**дагер-алиф** 0670) |
| 1:1/1:3 «рахим/-ман» | الرَّحِيمِ (обычная **кясра** U+0650) | الرَّح۪يمِ (**мелкий знак** ۪ U+06EA под ح) |
| 1:2 начало | الْحَمْدُ | اَلْحَمْدُ (помечена **хамза/фатха** на нач. алифе) |
| концы аятов | без знаков | вставлены **знаки вакфа** ۙ ۜ inline |

Т.е. вывод ресёрча ниже («разница ~90% шрифт») **НЕВЕРЕН в части текста** — dabt (огласовки/вакфы)
у Diyanet другой на уровне кодпоинтов (U+06EA вместо 0650, дагер-алиф, нач. хамза, вакфы). Владелец
прав: нужен ДРУГОЙ текст, не только шрифт. **Хорошая новость для нас:** различия только в диакритике/
знаках — СЛОВА (границы по пробелам) и их число те же → пословный `word_timeline` и forced (по
согласным) можно НЕ трогать: отображаем текст Diyanet, а выравниваемся по-прежнему на Tanzil.

**План П9 (уточнён):** (1) взять текст Diyanet (источник: `api.acikkuran.com` — Diyanet-based,
⚠️ лицензия CC BY-NC-SA = только некоммерч.; или офиц. Diyanet API `acikkaynakkuran-dev.diyanet.gov.tr`
по ключу) как ВТОРОЙ источник текста в quran.db (напр. колонка/таблица `text_diyanet`), пословно
сопоставим по числу слов с нашим; (2) в плеере переключатель «текст: Tanzil / Diyanet» (как шрифт);
(3) forced/word_timeline не меняем (те же слова). Шрифт Amiri Quran УЖЕ подключён и умеет U+06EA/дагер-алиф.

## Главный вывод ресёрча (ЧАСТИЧНО ОПРОВЕРГНУТ — см. выше)

Разница между мусхафом Diyanet и нашим Tanzil-текстом — **на ~90% это ШРИФТ + вёрстка**
(османский насх школы Хафыза Османа + разметка «аят-беркенар»: страница = целые аяты),
**а НЕ другой Unicode-текст букв**. Консонантный скелет (расм) у турецкого мусхафа Diyanet
**идентичен** нашему Tanzil Uthmani (наррация Hafs 'an Asim у всех одна; академический принцип —
каллиграфия может отличаться, расм нет). Реальные Unicode-отличия существуют, но только на уровне
**dabt** (огласовки, знаки вакфа-остановок, начертание хамзы/madd/sukun/wasl), и бо́льшая их часть —
это **разные глифы в пределах ОДНОГО кодпоинта** (решается шрифтом), а не другие combining-символы.

⚠️ Честная оговорка ресёрча: посимвольного Unicode-диффа «Diyanet vs Tanzil» в открытых источниках
не нашли → вывод «буквы совпадают, расходятся огласовки/вакфы» опирается на принцип неизменности
расма + литературу по dabt, не на прямое сравнение файлов. Т.е. возможно, владелец видит реальные
dabt-отличия — но даже они в большой мере шрифтовые.

## Источники ТЕКСТА (машинночитаемо)

- **Diyanet Açık Kaynak Kur'an API** — `https://acikkaynakkuran-dev.diyanet.gov.tr/` (ключ по GitHub-
  логину). Арабский Unicode + турецкий meal, текст утверждён советом Diyanet. Референс-апп
  `https://github.com/diyanet-bid/Kuran` (Next.js, **Apache-2.0**), опции «hat» Mushaf-ı Şerif /
  Medine hattı. **Лицензия на сами ДАННЫЕ не указана** — юр.чистоту распространения уточнять у Diyanet.
  Ближайшее к «тексту турецкого мусхафа», если понадобится.
- **QUL** (`qul.tarteel.ai/resources/quran-script`, `mushaf-layout`): скрипты Uthmani/IndoPak/Imlaei/
  DigitalKhatt/QPC; layouts KFGQPC/Indopak/Qatar. **Турецкого/Diyanet/ayet-berkenar НЕТ.**
- **Tanzil** (`tanzil.net/download`): Uthmani/Uthmani-minimal/Simple/Imlaei. Турецкого НЕТ. Лицензия:
  дословно копировать можно, менять текст нельзя, атрибуция обязательна. **Tanzil Uthmani = тот же
  османский расм, что у Diyanet на уровне букв.**
- `github.com/acik-kuran/acikkuran-api` — JSON пословно, но лицензия **CC BY-NC-SA (NonCommercial)** →
  для коммерции не годится. `github.com/fawazahmed0/quran-api` — есть турецкий ПЕРЕВОД Diyanet, но
  арабский текст стандартный. KFGQPC/QCF — саудовский мединский, не турецкий.

## Шрифты (веб, свободная лицензия, совместимы с пословной подсветкой)

- **Amiri Quran** — `fonts.google.com/specimen/Amiri+Quran`, **SIL OFL 1.1** (встраивание/коммерция ок).
  Классический насх линии Bulaq/Amiria (османская типографская традиция), полная поддержка
  коранических знаков, WOFF2. **Лучший практичный кандидат под «османский/турецкий вид».**
- **Scheherazade New** — SIL, OFL, насх, полная вокализация; визуально менее «османский», чем Amiri.
- ❌ **KFGQPC HAFS** — personal-use-only + саудовский вид. Не брать.
- ❌ **QUL glyph-шрифты (QPC)** — glyph-based, требуют своей layout-раскладки, **несовместимы с
  пословной подсветкой по обычному Unicode** → ломают привязку слов к таймлайну. Не брать.
- Официального свободного веб-шрифта Diyanet / аутентичного ayet-berkenar в Unicode нет (у Diyanet
  аутентика — страницы-картинки печатного мусхафа, не Unicode-шрифт).

## Критично для НАШЕЙ архитектуры

Пословная подсветка в плеере работает по **selectable Unicode-тексту, `<span>` на слово**. Значит
glyph-шрифты QPC, layout-мусхафы и картинки-страницы Diyanet **НЕ годятся** (ломают `word_timeline`).
Amiri Quran / Scheherazade New — настоящие Unicode-шрифты → совместимы без переделок форсед-данных.

## Рекомендация (порядок для прототипа)

1. **Дёшево:** оставить наш Tanzil-текст, подключить **Amiri Quran** (OFL, self-host woff2) как
   выбираемый шрифт в панели «⚙ Шрифт» плеера (там уже есть выбор шрифта+размера) → показать владельцу.
   Буквы и так совпадают с Diyanet; насх сразу даёт «мусхафный» вид, ближе к турецкому.
2. **Если владелец укажет на конкретные знаки** (вакфы/огласовки) — тогда текст с османским dabt из
   Diyanet API (взвесив лицензию + необходимость переalign'ить пословные данные под другой набор
   combining-символов).
3. **Ayet-berkenar** (страница=целые аяты) — фича вёрстки, дорого, для прототипа пропустить.

Смену Unicode-текста БУКВ делать НЕ нужно — расм и так правильный. Владельцу «скопировать классы/код
и не трогать оригинал» → на деле хватит ДОБАВИТЬ шрифт как опцию (переключение уже есть), без копии
текстовых классов. Если всё же понадобится diyanet-текст — тогда второй источник текста (switchable).
