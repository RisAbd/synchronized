# `data/quran.db` — схема переиспользуемого ассета

> quran.db задуман как **чистый переиспользуемый ассет** (используется и в других проектах владельца).
> Здесь — что где лежит и, главное, **как правильно рисовать басмалу**, чтобы не задвоить.
> Наполнение флагов: `tools/set_basmala_flags.py`. Маппинг редакций: `tools/build_edition_map.py`.

## Таблицы

### `surahs` — 114 сур
| колонка | смысл |
|---|---|
| `id`, `number` | id и номер суры (в этой БД совпадают, но не полагайся — привязка через `id`) |
| `title`, `verses_count`, `revelation_place`, `chronological_order` | метаданные суры |
| `bismillah_pre` | **флаг «дорисовать ﷽» для редакции `text` (Tanzil).** См. ниже. |
| `bismillah_pre_diyanet` | **флаг «дорисовать ﷽» для редакции `text_diyanet` (Diyanet).** См. ниже. |

### `surah_verses` — 6236 аятов
| колонка | смысл |
|---|---|
| `surah_id`, `number` | адрес аята (сура, номер аята) |
| `text` | текст аята, **редакция Tanzil** (имляи). Байт-в-байт как в издании — НЕ менять. |
| `text_diyanet` | текст аята, **редакция Diyanet** (тур. мусхаф, `api.acikkuran.com`, ⚠️ CC BY-NC-SA). Не менять. |
| `sacdah` | аят земного поклона |

### `edition_word_map` — карта слов Tanzil↔Diyanet (82634 строки)
`(surah, ayah, tanzil_wi, diyanet_wi)` — по строке на пару сопоставленных слов; `diyanet_wi=NULL`,
если слово Tanzil ни с чем не сматчено (вшитая басмала, отдельные токены-вакфы). Нужна, чтобы тайминги,
посчитанные по словам Tanzil, лечь на слова Diyanet при ином дроблении.

## Басмала — как рисовать (важно, тут был баг с задвоением)

**Обе редакции — стандарт Каир-1924 / куфийский счёт** (6236 аятов; басмала считается аятом только в
Фатихе; у Тавбы-9 басмалы нет). Нумерация аятов у редакций одинакова.

Различие — **в тексте**: Tanzil ВШИВАЕТ басмалу в `text` аята 1 (у 113 сур, кроме Тавбы), Diyanet — НЕТ.
Поэтому «нужно ли дорисовать отдельную ﷽ перед сурой» **зависит от редакции** → два флага:

| редакция (колонка текста) | флаг | значения | правило рендера |
|---|---|---|---|
| `text` (Tanzil) | `bismillah_pre` | **False у всех 114** | НЕ рисуй ﷽ отдельно — она уже в `text` аята 1 (а у Тавбы её нет вовсе) |
| `text_diyanet` (Diyanet) | `bismillah_pre_diyanet` | **True у 112; False у [1, 9]** | если True → нарисуй ﷽ перед аятом 1; у Фатихи (басмала=аят 1) и Тавбы (нет) — не рисуй |

Семантика флага = **«в тексте этой редакции басмалы нет → дорисуй ﷽ сам»**. Так потребитель никогда не
задвоит: при Tanzil-тексте флаг False (басмала в тексте), при Diyanet-тексте флаг True (текст чистый).

**Тавба (9): ﷽ не рисуется ни в одной редакции** (басмалы у суры нет) — оба флага False, есть assert-страховка.
