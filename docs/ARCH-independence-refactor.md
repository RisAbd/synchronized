# РЕФАКТОР: независимые источники + общий модуль матчинга (директива владельца 24.07)

> **Голос владельца 24.07 (резко, по делу).** Проблема: `w2v` наследует возвраты из `MMS`, `MMS`
> берёт диапазон из матчинга, матчинг — из `whisper`/`google`. «Империя на кривых распознавателях».
> Сравнивая прогоны, владелец думал, что смотрит РАЗНЫЕ подходы — а по сути один и тот же, где косяки
> одного затыкаются другим, набирая новые. **Требование: каждый источник ПОЛНОСТЬЮ независим,
> общая логика — в один переиспользуемый модуль, без наследования результатов. Проще, меньше слоёв.**

## Целевая архитектура

- **Источник (source)** — самодостаточный модуль: `audio → своя акустика → буквы/слова с временами`.
  Их РАВНОПРАВНЫЙ массив (никаких «типов» распознаватель/аллайнер):
  - `google` — Google STT → слова.
  - `whisper` — faster-whisper (tarteel ct2) → слова.
  - `mms` (быв. forced) — MMS-CTC → романизованные буквы/слова.
  - `w2v` — wav2vec2 (whisperx) → арабские буквы/слова.
  - `manual` — человек мышью (особый: истина).
- **Общий модуль `match_align`** (переиспользуется ВСЕМИ, не наследуется): `буквы/слова+времена +
  quran.db → sync_map` (какой аят где = матчинг к Корану + выравнивание слов + детект возвратов +
  snap к тишине). Один код, вызывается из каждого источника внутри. Сегодня это ЧАСТЬ `src/align.py`
  (матчинг ASR→Коран) — обобщить и на буквенный вход (MMS/w2v), чтобы аллайнеры сами определяли
  диапазон из СВОЕЙ акустики, а не брали у ASR.
- **Никаких кросс-зависимостей между источниками.** Каждый гоняется от аудио до word_timeline сам.
  Сравнение прогонов = сравнение РЕАЛЬНО разных подходов.

## Что сейчас нарушает (карта зависимостей)

- `pipeline._inherit_repeats` — **w2v тянет возвраты (rep-точки) из MMS sync-map.** ❌ убрать.
- `pipeline._forced_source` + `gpu_align.py` — **forced/w2v берут ДИАПАЗОН аятов из готового
  google/whisper.** ❌ источник должен определять диапазон сам (из своей акустики через match_align).
- `recognizers.ALIGNERS`/`is_aligner` + `tasks._maybe_forced`/`_maybe_w2v` — деление на типы и
  авто-пост-шаги «после ASR». ❌ уплостить: просто набор источников, каждый запускается сам.
- `models.active_run` — выбор по типам/приоритету. Упростить под плоский массив.

## Что УЖЕ хорошо (переиспользовать, не ломать)

- `src/align.py::align(transcript, quran)` — матчинг слов к Корану + сборка word_timeline. Это ядро
  общего `match_align` для СЛОВНОГО входа (google/whisper уже независимы через него).
- `falign._snap_bounds`, `_detect_repeats` (акустика), `_greedy_ctc` — кирпичи выравнивания/возвратов,
  переносятся в общий модуль.
- Возвраты на самоподобии аудио (проверено 24.07, дискриминирует повтор 0.26 vs норма 0.07–0.13) —
  универсальный детект возвратов, НЕ завязан на конкретную акустику → в общий модуль.

## Принцип-образец (владелец 24.07): плагины-по-соглашению + динамическая подгрузка

Владелец видел на другом своём проекте удачную структуру и предложил ПРИНЦИП (не копировать чужую
специфику — подходит ли, решаем отдельно; но для новой структуры полезно):

- **Папка с файлом на каждый тип.** Каждый тип-плагин — отдельный самодостаточный модуль, внутри —
  только его специфика.
- **Единый контракт ПО СОГЛАШЕНИЮ** (не базовый класс, не реестр): каждый модуль экспонирует
  функции с одинаковыми именами/сигнатурами. Пример из того проекта — уведомления, где каждый тип
  давал две функции: «посчитать, кого уведомить» и «отрендерить текст».
- **Динамическая подгрузка по имени**, вызов единообразный:
  ```python
  mod = importlib.import_module(f'{PKG}.{name}')   # name — имя типа
  result = mod.do_step(...)                         # одинаково для всех типов
  ```
- Новый тип = новый файл с теми же функциями. Никакой проводки. Duck-typing по имени модуля.

**Как это МОЖЕТ лечь на источники синхрона (гипотеза, проверить пригодность):** папка `sources/`,
модуль на источник (`google.py`/`whisper.py`/`mms.py`/`w2v.py`/`manual.py`), каждый — фикс-контракт
(напр. `recognize(audio) -> units`), лоадер грузит по имени, `pipeline` итерирует список. «Типов»
(aligner/recognizer) нет. Сегодня `recognizers.REGISTRY` — зачаток, но логика размазана по
`asr.py`/`falign.py`/`w2v_align.py`/`align.py`/`pipeline.py` с кросс-связями → в модули-источники.

**Общие шаги:** матчинг к тексту Корана, возможно и выравнивание — скорее ОБЩИЕ для всех (модуль
`match_align`, импортируется каждым источником). Либо общие только для источников, что выдают
ПОБУКВЕННО (mms/w2v): словесные (google/whisper) → общий словесный матчинг; буквенные → общий
буквенный матчинг + align. Различие источников — ТОЛЬКО в акустике/что выдают, остальное — общий код.

**Директива по w2v: делать С НУЛЯ, сразу нормально** — НЕ «временно беру отсюда, отсюда» (не
наследовать промежуточные результаты MMS). Переписать `w2v` источником с чистого листа под единый
контракт + общий `match_align`.

## ✅ Де-рискинг 24.07: КАК w2v определяет диапазон САМ (без ASR) — доказано на rec7

Владелец (голос 24.07, жёстко): w2v НЕ использует НИЧЕГО от google/whisper/forced — ни диапазон,
ни windows, ни возвраты. Только своя акустика. Значит нужен «свой распознаватель» из w2v-эмиссий.
Прогон эмпирики (эмиссии в `work/rec7_w2v_emis.npy`, пробы `work/probe_*`):

1. **wav2vec2 (whisperx align-модель = полноценная CTC ASR) отдаёт арабские буквы+харакаты**
   (словарь 51: буквы 6..45, харакаты 46..50, `<pad>`=blank=0). Концепт есть.
2. **Greedy-декод БЕДЕН: 84% кадров = blank** (мелодичный таджвид далёк от обучающей речи модели →
   «молчит» на распевах, буквы в 14% кадров). Декод rec7 = 1053 símb на 255с. Словесный матчинг
   (`align.py`) на нём = **cov 0.0**; char-level k-грамм-матчинг/монотонная цепочка — сура 6
   всплывает как кандидат, но диапазон грубый/шумный. Greedy борется с разреженностью и проигрывает.
3. **✅ РЕШЕНИЕ — CTC-forward-скоринг (не декод!).** Считаем logP(эмиссии | текст-кандидат) CTC-forward'ом
   в лог-пространстве (numpy, БЕЗ GPU, на сохранённых эмиссиях), нормируем на длину меток. CTC
   МОДЕЛИРУЕТ blank'и (не выбрасывает) → разреженность не мешает. Истинное окно = максимум score.
   **rec7 (истина 6:95-103):** norm/char −3.803 (ЛУЧШИЙ) vs 6:95-110 надмн. −3.836, 6:90-103 −4.377,
   6:60-78 −4.383, 2:1-20 −4.523, 6:1-9 −5.859, 6:99-103 подмн. −7.211. **Истинный диапазон
   выигрывает чётко.** Проба — `work/probe_ctc_score.py` (векторизованный CTC-forward по S).

**→ Архитектура независимого w2v (строим):** audio → wav2vec2 эмиссии (вся запись) → (a) грубый
кандидат-регион (greedy-декод + k-граммы → сура/окрестность) → (b) CTC-скоринг скользящих окон
корпуса вокруг кандидата → ТОЧНЫЙ диапазон surah:ayah → (c) force-align этого диапазона (whisperx)
→ границы слов → (d) СВОИ возвраты из своей акустики (см. ниже). Ноль данных от других источников.

### Возвраты w2v из своей акустики (де-рискано на rec7 6:101 «وخلق كل شيء ×2»)

## 🔴 СЛЕДУЮЩИЙ ШАГ (24.07, курс владельца): выкинуть whisperx, свой forced-align по эмиссиям

Владелец (голос): «зачем wave2vec нужен whisperx? делай сам на себе». WhisperX = обёртка (грузит
wav2vec2-модель + даёт CTC-forced-align). Мозг — 100% wav2vec2, но именно whisperx-овская функция
выравнивания ЛОМАЕТ мелодичный хвост (rec7: collapsed_words=14, 6:102:10..103:7 в один момент 210.16с;
патч «разложить слипшееся» в w2v_align НЕ помог — клуппинг внутри whisperx). Решение:
- **Заменить `whisperx.align` на СВОЙ CTC-forced-align по wav2vec2-эмиссиям** (эмиссии уже считаю в
  `w2v_align.emissions`; CTC-forward уже есть в `w2v_range.ctc_logprob`). Нужен Viterbi-вариант:
  расширенная последовательность меток (blank между буквами) → argmax-путь с backpointer'ами по
  эмиссиям → кадр на каждую метку → границы слов (t0/t1). Мелодичные (blank) участки распределять
  честно (интерполяция между акустически-опорными буквами, не сваливать в точку).
- Результат: `w2v_align` без whisperx (только модель через `load_align_model` для эмиссий, или прямой
  HF Wav2Vec2ForCTC). Чистый wav2vec2. Клуппинг решается в корне.
- Уже независимо и работает: диапазон (`w2v_range`, difflib по буквам — rec7 6:95-103, rec5 25:63-77),
  self-derived старты аятов (difflib decode↔текст), возвраты (`w2v_repeats`), гейт `_forced_source`
  убран. Осталось: своё выравнивание (этот шаг) + прогон всех записей через celery-флоу + аудит.

## Принцип на будущее (записать и держать)

Не строить источник на выходе другого источника. Общее — только в общем модуле, импортом. Тогда
каждая новая идея улучшает КОНКРЕТНЫЙ источник, а не тянет чужие косяки. (Мой прошлый подход —
слой на слой, наследование результатов — привёл к «каше»; это зафиксировано как урок.)
