## Режим работы

- **Читай `docs/BACKLOG.md` ПЕРВЫМ.** Там актуальные статусы, «Next» и указки владельца.
- **Работаем автономно, без переспрашиваний.** Владелец часто отсутствует, даёт короткие указки
  (нередко голосом `.ogg` → skill `audio-task`) и периодически говорит «продолжай развитие».
- **Коммить И ПУШИТЬ на каждом завершённом шаге / логической точке — и продолжай.** Коммит и пуш
  ВСЕГДА вместе, автоматически, без подтверждений. Не жди разрешения, не копи гору изменений.
  Дошёл до осмысленного состояния → `git commit` с внятным сообщением → `git push` → дальше.
- **`/self-compact` после КАЖДОГО закрытого пункта беклога** (голос владельца 11.07): когда пункт
  реально завершён — код готов, `git commit`+`git push` сделаны, И все записи обновлены (BACKLOG
  прогресс, «Немедленная память», memory) → делай `/self-compact` и продолжай. Ритм ~15–20 мин
  активной работы. НЕ компактить на середине задачи — только на завершённой логической точке.
  Если пункт закрылся быстрее ~15 мин — НЕ компактить сразу, накопи и сделай `/self-compact`
  после следующего закрытого пункта (не дробить компакты на мелочь).
- **ВСЕГДА МОЛОТИ. Не останавливайся спрашивать «что дальше».** Закончил задачу → коммит+пуш →
  СРАЗУ берёшь следующий пункт. Нет явной указки / не осталось (по-твоему) важного — бери
  **следующий пункт из `docs/BACKLOG.md`** (верхний невыполненный из «Next»/🔴/🟡) и делай.
  Простой в ожидании вопросов владельца = ошибка. Останавливаться можно только когда реально
  нечего взять из беклога (тогда — предложить/наполнить беклог), либо нужен ввод, который есть
  ТОЛЬКО у владельца (напр. фидбек с реального телефона).
